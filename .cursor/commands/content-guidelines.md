# Content Writing Guidelines

## General Principles

1. **Clarity and Conciseness**
   - Write in clear, professional language
   - Avoid jargon unless necessary (then explain)
   - Keep paragraphs short and focused
   - Use bullet points for lists

2. **Australian English**
   - Use Australian spelling throughout
   - Examples: "organise" not "organize", "colour" not "color"
   - Use Australian date format (DD/MM/YYYY)
   - Use 24-hour time format

3. **Accessibility**
   - Use plain language
   - Write descriptive headings
   - Use proper heading hierarchy
   - Ensure content is screen reader friendly

## Case Study Structure

### Required Sections

1. **Title and Metadata**
   - Clear, descriptive title
   - Client name
   - Industry
   - Duration and year
   - Challenge statement

2. **Challenge**
   - Describe the problem or opportunity
   - Explain context and constraints
   - Use specific details, not generic statements

3. **Solution/Approach**
   - Explain the methodology
   - Describe key activities
   - Show the process used
   - Include stakeholder involvement

4. **Implementation**
   - Detail how the solution was delivered
   - Describe tools and techniques used
   - Explain any technical aspects
   - Note any challenges overcome

5. **Outcomes**
   - Quantify results with metrics
   - Describe qualitative impacts
   - Show before/after comparisons
   - Include client feedback if available

6. **Key Learnings**
   - What worked well
   - What would be done differently
   - Insights gained
   - Applicable lessons

### Writing Tips

- **Start with the challenge** - Set context immediately
- **Use metrics** - Numbers are more compelling than adjectives
- **Show, don't tell** - Use examples and specifics
- **Be honest** - Include challenges and learnings
- **Keep it focused** - One main story per case study

## Homepage Content

### Hero Section
- Clear value proposition
- Compelling headline
- Brief subheading
- Strong call-to-action

### Service Offerings
- 3-4 key service areas
- Clear descriptions
- Benefit-focused language
- Link to detailed pages if available

### Approach/Methodology
- Describe the process
- Show expertise
- Build confidence
- Use visual elements if possible

## About Page

### Structure
1. **Introduction** - Who you are and what you do
2. **Background** - Experience and expertise
3. **Approach** - How you work
4. **Values** - What matters to you
5. **Contact** - How to get in touch

### Tone
- Professional but approachable
- Authentic and genuine
- Focus on value delivered
- Show personality

## Contact Page

### Required Elements
- Clear contact form
- Alternative contact methods
- Response time expectations
- What happens after submission

### Form Fields
- Name (required)
- Email (required)
- Company/Organisation (optional)
- Message (required)
- How did you hear about us (optional)

## SEO Guidelines

### Meta Descriptions
- 150-160 characters
- Include primary keyword
- Compelling and clear
- Include call-to-action

### Headings
- Use H1 for page title only
- Use H2 for main sections
- Use H3 for subsections
- Maintain hierarchy

### Keywords
- Use naturally in content
- Include in frontmatter
- Focus on user intent
- Avoid keyword stuffing

## Image Guidelines

### Alt Text
- Describe the image content
- Include context if relevant
- Keep it concise
- Don't start with "Image of..."

### File Naming
- Use descriptive names
- Use kebab-case
- Include context
- Example: `antarctic-safety-protocol-diagram.webp`

### Optimization
- Use WebP format when possible
- Optimize file sizes
- Use appropriate dimensions
- Consider lazy loading

## Tone and Voice

### Characteristics
- **Professional** - But not corporate jargon
- **Clear** - Easy to understand
- **Confident** - Show expertise without bragging
- **Approachable** - Friendly and accessible
- **Results-focused** - Emphasize outcomes

### Avoid
- Marketing speak
- Buzzwords
- Overly technical language (unless necessary)
- Vague statements
- Exaggeration

## Formatting

### Markdown Best Practices
- Use proper heading hierarchy
- Use lists for multiple items
- Use blockquotes for testimonials
- Use code blocks for technical content
- Use emphasis (bold/italic) sparingly

### Links
- Use descriptive link text
- Don't use "click here"
- Make links clear and obvious
- Test all links before publishing

## Review Checklist

Before publishing content:

- [ ] Australian English spelling checked
- [ ] Grammar and punctuation reviewed
- [ ] All facts verified
- [ ] Metrics and numbers accurate
- [ ] Links tested and working
- [ ] Images optimized and have alt text
- [ ] SEO metadata complete
- [ ] Accessibility checked
- [ ] Mobile readability verified
- [ ] Professional tone maintained

## Examples

### Good Case Study Opening

> **Challenge:** The University of Canberra needed to transition 2,200+ courses to online delivery within 4 weeks during the COVID-19 pandemic while maintaining educational quality and supporting 800+ academic staff through rapid digital transformation.

### Poor Case Study Opening

> **Challenge:** We helped a university with digital transformation during a difficult time.

### Good Metric

> Achieved 92% student satisfaction with digital learning and reduced IT support tickets by 45%.

### Poor Metric

> Improved satisfaction and reduced support issues significantly.
