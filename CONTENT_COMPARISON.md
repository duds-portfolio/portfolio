## Content Structure Differences

### Case Study Format Comparison

#### Local & portfolio_v3 Format (`.md`)
- Uses standard Markdown frontmatter
- More detailed metadata including:
  - `keywords`
  - `ogImage`
  - `category`
  - `featured` flag
- More comprehensive content structure with detailed sections
- Example: `asic-fit-and-proper-person-test.md` has extensive service design documentation

#### dale_rogers_portfolio Format (`.mdx`)
- Uses MDX frontmatter (similar structure)
- Different metadata fields:
  - `pubDate` instead of `year`
  - `tags` as array
  - `coverImage` instead of `ogImage`
  - `featured` flag
  - `client`, `industry`, `duration`, `challenge`, `solution`, `results` fields
- More concise content structure
- Example: `a-better-fit-and-proper-person-test.mdx` is shorter and more focused

### Article Format Comparison

#### Local Format (`.md`)
- Standard Markdown frontmatter
- Fields: `title`, `description`, `pubDate`, `authorName`, `authorImage`, `image`, `tags`
- Content in standard Markdown

#### dale_rogers_portfolio Format (`.mdx`)
- MDX frontmatter
- Fields: `title`, `description`, `pubDate`, `author`, `tags`, `coverImage`
- Content in MDX (can include JSX components)

## File Format Notes

- **`.md`**: Standard Markdown - simpler, more portable
- **`.mdx`**: Markdown with JSX - allows React components, more powerful but requires MDX processing

Current Local portfolio uses `.md` format, which is simpler and more compatible with standard static site generators.
