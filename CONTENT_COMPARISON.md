  - `ogImage`

- Example: `asic-fit-and-proper-person-test.md` has extensive service design documentation

#### dale_rogers_portfolio Format (`.mdx`)
- Uses MDX frontmatter (similar structure)
- Different metadata fields:
  - `pubDate` instead of `year`
  - `tags` as array
  - `coverImage` instead of `ogImage`

- More concise content structure
- Example: `a-better-fit-and-proper-person-test.mdx` is shorter and more focused

#### Local Format (`.md`)

- Fields: `title`, `description`, `pubDate`, `authorName`, `authorImage`, `image`, `tags`
- Content in standard Markdown

#### dale_rogers_portfolio Format (`.mdx`)
- MDX frontmatter
- Fields: `title`, `description`, `pubDate`, `author`, `tags`, `coverImage`
